# Vacation Rental Booking Automation

A production-ready Python system that automatically extracts vacation rental booking data from Gmail inboxes and syncs it to PostgreSQL. The system supports multiple platforms including Vrbo, Airbnb, and Booking.com.

---

## 🚀 Development & Branching Rules

To maintain the integrity of the production environment, all developers must follow these rules:

### 1. Main Branch Protection
- The **`main`** branch is reserved for production-ready, stable code.
- **Direct commits to `main` are strictly prohibited.**
- `main` is used only for Pull Requests (PRs).

### 2. Feature Development
- All new features, bug fixes, or experiments must be done in a **sub-branch**.
- Branch naming convention: `feature/your-task` or `fix/issue-description`.
- Create a sub-branch:
  ```bash
  git checkout -b feature/new-api-endpoint
  ```

### 3. Deployment & PR Workflow
Follow this exact sequence for every change:
1. **Develop** the feature in your sub-branch.
2. **Deploy** the sub-branch to the test environment.
3. **Verify** that the deployment is working correctly and passing all checks.
4. **Generate PR**: Only *after* a successful deployment and verification, you are allowed to generate a Pull Request to merge the sub-branch into `main`.
5. **Merge**: Once the PR is reviewed and approved, it will be merged into `main`.

## 🛠️ Features

- **Email Reader**: Connects to Gmail via IMAP to fetch booking confirmation emails.
- **Booking Parser**: Extracts structured data from emails using HTML parsing and regex.
- **PostgreSQL Sync**: Uploads bookings to a local or remote Postgres DB with duplicate detection.
- **Provider Response Tracker**: Real-time tracking of accept/reject responses from crews and service providers.
- **CLI Interface**: Command-line tool with various options for processing.
- **Automated Notifications**: 
    - **Email**: Powered by **SendGrid** for professional guest communication.
    - **SMS/WhatsApp**: Powered by **Twilio** for instant staff and guest alerts.

## ⚙️ Installation & Setup

1. **Clone the repository**:
   ```bash
   git clone <repository-url>
   cd Email-parser
   ```

2. **Create a virtual environment**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Database Migrations**:
   The project uses Alembic for database schema management. Run the following to apply migrations:
   ```bash
   alembic upgrade head
   # OR use the helper script
   ./run_migrations.sh
   ```

5. **Set up environment variables**:
   Create a `.env` file in the root directory:
   ```env
   # Database
   DATABASE_URL=postgresql+asyncpg://postgres:yourpasswrd@localhost:5432/str_command_center
   
   # API & URLs
   API_BASE_URL=http://127.0.0.1:8001
   PORT=8001
   
   # Gmail
   GMAIL_EMAIL=your-email@gmail.com
   GMAIL_PASSWORD=your-app-password

   # Notifications
   SENDGRID_API_KEY=your_sendgrid_key
   TWILIO_ACCOUNT_SID=your_twilio_sid
   TWILIO_AUTH_TOKEN=your_twilio_token
   ```

## 🗄️ Database Schema Requirements

Ensure your production database has the following updates:

### **1. Task Responses Table**
```sql
CREATE TABLE task_responses (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    task_id TEXT NOT NULL,
    task_type TEXT NOT NULL,
    response TEXT NOT NULL,
    task_name TEXT,
    person_name TEXT,
    task_date_time TEXT
);
```

### **2. Status Columns**
Add `status` (TEXT, default 'pending') to existing tables:
- `cleaning_tasks`
- `booking_service`

### **3. Property Columns**
Add to `properties` table:
- `base_price` (NUMERIC, default 0)
- `bedrooms` (INTEGER, default 0)

### **4. Booking Table Structure**
The main table for storing extracted reservation data:

| Column | Type | Description |
|--------|------|-------------|
| `reservation_id` | TEXT | Primary Key (e.g., Airbnb/Vrbo ID) |
| `platform` | TEXT | Source (airbnb, vrbo, booking) |
| `guest_name` | TEXT | Name of the guest |
| `guest_phone` | TEXT | Guest contact number |
| `guest_email` | TEXT | Guest email address |
| `check_in_date` | TIMESTAMPTZ | Arrival date |
| `check_out_date` | TIMESTAMPTZ | Departure date |
| `property_id` | TEXT | Internal property identifier |
| `property_name` | TEXT | Name of the vacation rental |
| `number_of_guests`| INTEGER | Total guest count |
| `total_amount` | DOUBLE PRECISION| Total reservation cost |
| `currency` | TEXT | Currency code (USD, etc.) |
| `booking_date` | TIMESTAMPTZ | Date when booking was made |
| `nights` | INTEGER | Number of nights stayed |
| `email_id` | TEXT | Unique ID of the source email |
| `raw_data` | JSON | Original parsed data fields |
| `created_at` | TIMESTAMPTZ | Record creation timestamp |
| `updated_at` | TIMESTAMPTZ | Last record update timestamp |
